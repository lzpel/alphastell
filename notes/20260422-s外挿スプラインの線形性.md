# s 外挿スプラインの線形性 — 係数先と座標先は等価

first_wall を s=1.08 で生成するときの「スプライン外挿」について、(A) Fourier 係数を先に補間する現在の方式と、(B) 各グリッド点で座標 (R,Z) を組み立ててからスプラインする方式が、数学的に等価であることを確認するメモ。parastell との ~3% ズレの原因がアーキテクチャ選択ではなく境界条件と幾何定義にあることを合わせて整理する。

## 前提のデータ構造

- `s_grid`: 等間隔 201 点、`s_grid[i] = i / (ns-1)` で `i = 0..200`、末端 `s_grid[200] = 1.0`
- `rmnc[i][k]`: s 軸 i 番目の磁束面における k 番目のモードの R 振幅 (`ns × mnmax`, 201 × 179)
- `zmns[i][k]`: 同じく Z の sin 成分
- `mode_poloidal[k]` = m_k, `mode_toroidal[k]` = n_k (整数、f64 で保持)

Fourier 和は
$$
R(\theta,\varphi,s) = \sum_{k=0}^{m_{\max}-1} \mathrm{rmnc}(s,k)\,\cos(m_k\theta - n_k\varphi)
$$
$$
Z(\theta,\varphi,s) = \sum_{k=0}^{m_{\max}-1} \mathrm{zmns}(s,k)\,\sin(m_k\theta - n_k\varphi)
$$

離散 s 点ではファイルから直接取れるが、s=1.08 のような外挿点では s 方向に補間が要る。

## (A) 係数先 — 現在 & parastell 方式

1. 各モード `k` について、201 点 `{(s_i, rmnc[i][k])}_{i=0..200}` を 3 次スプラインで繋ぎ、`s*=1.08` で評価する:
   $$
   \widetilde{\mathrm{rmnc}}(s^*, k) = \sum_{i=0}^{n_s-1} L_i(s^*)\,\mathrm{rmnc}[i][k]
   $$
   ここで `L_i(s)` は s グリッドと境界条件で一意に決まる基底関数 (= スプライン補間が `y_i` に対して線形であることの表現)。

2. 得られた `mnmax` 個の補間係数で Fourier 和:
   $$
   R(\theta,\varphi,s^*) = \sum_{k} \widetilde{\mathrm{rmnc}}(s^*,k)\,\cos(m_k\theta - n_k\varphi)
   $$

計算コスト: **mnmax = 179 本**のスプラインを 1 度構築、残りは cos/sin 和のみ。M_TORO × N_POLO = 15360 回の (θ,φ) サンプリングに使い回せる。

`src/vmec.rs::interp_coeffs_at_s` (s 外挿) + `src/vmec.rs::eval_rz` (Fourier 和) の構成がこれ。

## (B) 座標先 — 提案されていた別方式

1. s グリッド上の各点 `s_i` で Fourier 和を先に計算:
   $$
   R_i(\theta,\varphi) = \sum_{k} \mathrm{rmnc}[i][k]\,\cos(m_k\theta - n_k\varphi)
   $$

2. その 201 点 `{(s_i, R_i(\theta,\varphi))}` を 3 次スプラインに通して `s*=1.08` を外挿:
   $$
   R(\theta,\varphi,s^*) = \sum_{i=0}^{n_s-1} L_i(s^*)\,R_i(\theta,\varphi)
   $$

計算コスト: **各 (θ,φ) ごとに 1 本**のスプラインが必要 → M_TORO × N_POLO = 15360 本。構築回数が約 86 倍多い。

## (A) と (B) が等価である証明

`L_i(s*)` は (A) で「rmnc の s 方向」、(B) で「R の s 方向」に使われるが、s グリッドと境界条件が共通なので**同じ基底関数**。この前提のもとで (B) の式を展開すると:

$$
R(\theta,\varphi,s^*) = \sum_i L_i(s^*)\,R_i(\theta,\varphi)
= \sum_i L_i(s^*)\,\sum_k \mathrm{rmnc}[i][k]\,\cos_k(\theta,\varphi)
$$

Σ の順序を入れ替えて

$$
= \sum_k \underbrace{\left(\sum_i L_i(s^*)\,\mathrm{rmnc}[i][k]\right)}_{=\,\widetilde{\mathrm{rmnc}}(s^*,k)}\,\cos_k(\theta,\varphi)
= \sum_k \widetilde{\mathrm{rmnc}}(s^*,k)\,\cos_k(\theta,\varphi)
$$

となり、(A) の式と**完全に一致**する。

キーになっている性質は 2 つ:

- Fourier 和が **k 方向に線形** (`Σ_k a_k · cos_k` の形)
- スプラインが **y 値に線形** (`S(x*) = Σ_i L_i(x*) · y_i`)

両方線形なので Σ_i と Σ_k を交換できて同じ値になる。

## 結論: parastell との差は広がらない

同じスプライン方式 (同じ `s_grid`、同じ境界条件) を使う限り、(A) と (B) の結果は数値誤差 (倍精度で 10⁻¹⁵ 程度) までしか違わない。したがって:

| 比較 | 結果の一致 | parastell との差 |
|---|---|---|
| 本 repo (A, natural) vs 本 repo (B, natural) | ✅ 一致 | 同じ (~3%) |
| 本 repo (A, natural) vs parastell (A, not-a-knot) | ❌ 境界条件差 | ~3% |
| 本 repo (B, natural) vs parastell (A, not-a-knot) | ❌ 境界条件差 | ~3% (オーダ同じ) |

(A)↔(B) の切り替えは **parastell との差を広げも縮めもしない**。3% は以下 2 点に起因する別レイヤーの話:

1. **境界条件の違い** — 本 repo `NaturalSpline` は `natural` (両端 2 階微分 = 0)、parastell は scipy `CubicSpline` のデフォルト `not-a-knot` (両端近傍で 3 階微分連続)。s=1.08 の外挿領域で数 % 差が出る。
2. **法線/厚み定義の違い** — parastell は 2D poloidal 断面で offset、本 repo (cadrum) は 3D surface で `Solid::shell`。first_wall の thickness 方向で数 % 差。

この合計で約 3%。`makefile:40-42` の tolerance 5% はこれを許容する値。

## コスト比較

| | スプライン構築 | Fourier 和 |
|---|---|---|
| (A) 係数先 | mnmax = **179 本** | M_TORO·N_POLO = 15360 回 |
| (B) 座標先 | M_TORO·N_POLO = **15360 本** | M_TORO·N_POLO·n_s ≈ 3,000,000 回 |

結果が同じで (B) が ~100 倍遅いので、(A) を採る。

## 自然スプラインの外挿挙動 (参考)

`NaturalSpline` は両端で 2 階微分 = 0 を課しているので、最終区間 `[s_{n-2}, s_{n-1}] = [0.995, 1.000]` の係数は

- `c_{n-2} = M_{n-2} / 2` (内部で決まる)
- `d_{n-2} = (M_{n-1} - M_{n-2}) / (6 h_{n-2}) = -M_{n-2} / (6 h)` (M_{n-1}=0 なので)

評価時 (`src/vmec.rs::eval`):

```rust
if x >= self.xs[n - 1] { idx = n - 2 }   // s*=1.08 はこのブランチ
let dx = x - self.xs[idx];               // 1.08 - 0.995 = 0.085
a[idx] + b[idx]·dx + c[idx]·dx² + d[idx]·dx³
```

dx = 0.085 と小さく、2 階微分が 0 に引かれているので外挿は**ほぼ線形**に振る舞う。これが「natural スプラインは端点で暴れにくい」という性質。
